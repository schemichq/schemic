surreal-zod lets you author your **SurrealDB** schema in TypeScript using a drop-in
for Zod (`sz.*`). From that single source of truth it generates the SurrealQL DDL,
infers your end-to-end types, and runs safe, reviewable migrations.

## Why schema-as-code

- **One source of truth** — your schema lives in TypeScript, in git.
- **Generated DDL** — `DEFINE TABLE / FIELD / INDEX / EVENT / FUNCTION / ACCESS`.
- **End-to-end types** — `App<>` (app-side) and `Wire<>` (DB-wire-side).
- **Safe migrations** — incremental `ALTER`, drift detection, rollbacks.
