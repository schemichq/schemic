---
title: Live queries
description: Decode live subscription payloads exactly like query rows.
icon: Radio
---

There's no special live API — a subscription payload is just a row, so decode it exactly
like a query result.

```ts twoslash
import { sz, table } from "surreal-zod";
import { Surreal, Table } from "surrealdb";
declare const db: Surreal;
const User = table("user", {
  id: sz.string(),
  name: sz.string(),
  createdAt: sz.datetime(),
});
// ---cut---
const subscription = await db.live(new Table("user"));

subscription.subscribe((msg) => {
  if (msg.action === "DELETE" || msg.action === "KILLED") return;
  const user = User.decode(msg.value); // a decoded App<User> — RecordId, Date, …
  //    ^?
});
```

Because `decode` validates as it maps, a malformed payload surfaces as a `ZodError` instead
of silently flowing through as `unknown`.

<Callout type="info">
  A typed query + live layer — where results are decoded automatically and `SELECT`
  projections are inferred — is planned in `surreal-zod/orm`. See the
  [Roadmap](/docs/roadmap/query-builder).
</Callout>
