`sz.object({ ... })` builds a nested SurrealQL `object`, and surreal-zod looks *through* it
so every nested field keeps its own DDL metadata and create-optionality.

```ts twoslash
import { sz, table, defineTable } from "surreal-zod";
import { surql } from "surrealdb";
// ---cut---
const Project = table("project", {
  name: sz.string(),
  settings: sz.object({
    isPublic: sz.boolean().$default(surql`false`),
    defaultView: sz.enum(["list", "board"]).$default("list"),
  }),
});

const ddl = defineTable(Project);
```

```sql
DEFINE TABLE project TYPE NORMAL SCHEMAFULL;
DEFINE FIELD name ON TABLE project TYPE string;
DEFINE FIELD settings ON TABLE project TYPE object;
DEFINE FIELD settings.isPublic ON TABLE project TYPE bool DEFAULT false;
DEFINE FIELD settings.defaultView ON TABLE project TYPE string DEFAULT "list";
```

## Create-optionality recurses

A nested field with a DB `$default` (or `$value(..., { optional: true })`) is
**create-optional** — omit it and the DB fills it — exactly like a top-level defaulted
field. So you can pass a *partial* nested object on create:

```ts twoslash
import { sz, table } from "surreal-zod";
import { surql } from "surrealdb";
const Project = table("project", {
  name: sz.string(),
  settings: sz.object({
    isPublic: sz.boolean().$default(surql`false`),
    defaultView: sz.enum(["list", "board"]).$default("list"),
  }),
});
// ---cut---
Project.encode({ name: "Launch", settings: { defaultView: "board" } });
// the DB fills settings.isPublic
```

On the **decoded** side those nested fields stay **required** in `App<T>` — a stored row
has them.

## `encode` (CONTENT) vs `encodePartial` (MERGE)

`encode` builds a `CONTENT` payload. `encodePartial` builds a **deep-partial** `MERGE`
payload — every nested key is optional — which mirrors SurrealDB's `MERGE`, that
**recursively deep-merges** (siblings are preserved at every level):

```ts twoslash
import { sz, table } from "surreal-zod";
import { Surreal, surql } from "surrealdb";
declare const db: Surreal;
declare const id: string;
const Project = table("project", {
  name: sz.string(),
  settings: sz.object({
    isPublic: sz.boolean().$default(surql`false`),
    defaultView: sz.enum(["list", "board"]).$default("list"),
  }),
});
// ---cut---
// only settings.defaultView changes; settings.isPublic is preserved by MERGE
await db.query(surql`UPDATE ${id} MERGE ${Project.encodePartial({
  settings: { defaultView: "list" },
})}`);
```

<Callout type="warn">
  The library only **builds the payload** — you choose the statement. Sending a *partial*
  payload with `CONTENT` **replaces** the record (unsupplied fields are dropped). Use
  `MERGE` for partial writes.
</Callout>

## Atomic (object-level) validation

Only objects built with `sz.object` are flattened and recursed. A field that holds a **raw,
refined `z.object`** is validated **atomically** — provide it whole, and its object-level
`refine` runs:

```ts twoslash
import { table } from "surreal-zod";
import { z } from "zod";
// ---cut---
table("booking", {
  // validated all-or-nothing; the refine runs on the whole object
  range: z.object({ from: z.number(), to: z.number() }).refine((r) => r.from <= r.to),
});
```

That's the built-in escape when you want all-or-nothing / object-level checks.
