The whole JS⇄DB mapping rides on Zod's two codec channels:

- the **decoded** side (`z.output`) is your **app** type;
- the **encoded** side (`z.input`) is the **DB wire** type.

`decode` reads from the DB; `encode` writes to it.

## Reading: `decode`

`decode(row)` turns a raw DB row into your typed app object — `DateTime` → `Date`,
`Uuid` → `string`, ids → `RecordId`, and so on.

```ts twoslash
import { sz, table } from "surreal-zod";
import { Surreal, surql } from "surrealdb";
declare const db: Surreal;
const User = table("user", {
  id: sz.string(),
  name: sz.string(),
  createdAt: sz.datetime(),
});
// ---cut---
const [rows] = await db.query<[unknown[]]>(surql`SELECT * FROM user`);
const user = User.decode(rows[0]);
//    ^?
```

## Writing: `encode` / `encodePartial`

Both build a payload from your app object and are **create / patch-shaped**: DB-filled
fields (`$default`, `id`) are *optional*, absent keys are omitted, and each provided value
is validated.

- Pair **`encode`** with **`CONTENT`** (create / full replace).
- Pair **`encodePartial`** with **`MERGE`** (partial update).

```ts twoslash
import { sz, table } from "surreal-zod";
import { Surreal, surql } from "surrealdb";
declare const db: Surreal;
const User = table("user", {
  id: sz.string(),
  name: sz.string(),
  email: sz.email(),
  status: sz.string().$default("pending"),
  createdAt: sz.datetime().$default(surql`time::now()`).$readonly(),
});
declare const id: string;
// ---cut---
// create
await db.query(surql`CREATE user CONTENT ${User.encode({
  name: "Ada",
  email: "ada@example.com",
})}`);

// partial update
await db.query(surql`UPDATE ${id} MERGE ${User.encodePartial({ status: "active" })}`);
```

The return value is a typed `Partial<Wire<T>>` — codec fields are their **wire** type:

```ts twoslash
import { sz, table } from "surreal-zod";
import { surql } from "surrealdb";
const User = table("user", {
  id: sz.string(),
  name: sz.string(),
  email: sz.email(),
  createdAt: sz.datetime().$default(surql`time::now()`).$readonly(),
});
// ---cut---
const payload = User.encode({ name: "Ada", email: "ada@example.com" });
payload.createdAt;
//      ^?
```

### The type guards your input

Required fields are enforced, and readonly / id keys are rejected from updates — at compile
time:

```ts twoslash
// @errors: 2345 2353
import { sz, table } from "surreal-zod";
import { surql } from "surrealdb";
const User = table("user", {
  id: sz.string(),
  name: sz.string(),
  email: sz.email(),
  createdAt: sz.datetime().$default(surql`time::now()`).$readonly(),
});
// ---cut---
User.encode({ name: "Ada" });                 // email is required
User.encodePartial({ createdAt: new Date() }); // createdAt is readonly
```

## Safe and async variants

Every write throws a `ZodError` on invalid input. For the non-throwing form use `safeEncode`
/ `safeEncodePartial`, which return Zod's `{ success, data } | { success, error }` envelope
(all field errors aggregated). Each has an async twin for async refinements.

```ts twoslash
import { sz, table } from "surreal-zod";
const User = table("user", { id: sz.string(), email: sz.email() });
// ---cut---
const res = User.safeEncode({ email: "not-an-email" });
if (res.success) {
  res.data;
//    ^?
} else {
  res.error.issues; // every field error, aggregated
}

await User.safeEncodeAsync({ email: "a@b.co" });
```

The same set exists for reads: `decode` / `safeDecode` / `decodeAsync` / `safeDecodeAsync`.
(`parse*` are kept as `@deprecated` aliases of `decode*` for `z`-API familiarity.)

## Field-level codecs

The same channels work on a single field — handy for one-off conversions:

```ts twoslash
import { sz } from "surreal-zod";
import type { DateTime } from "surrealdb";
declare const dbValue: DateTime;
// ---cut---
sz.datetime().decode(dbValue);    // -> Date
sz.datetime().encode(new Date()); // -> DateTime
sz.uuid().encode("0190b6e0-0000-7000-8000-000000000000"); // -> Uuid
```

## Escape hatch: the raw codec

`encode` / `encodePartial` are conveniences. If you need the **raw, full-object codec**
(no create-shaping), it's just Zod:

```ts twoslash
import { sz, table, type App } from "surreal-zod";
import { z } from "zod";
const User = table("user", { id: sz.string(), name: sz.string() });
declare const fullUser: App<typeof User>;
// ---cut---
const wire = z.encode(User.object, fullUser);
```
