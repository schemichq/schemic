This walkthrough defines a `user` table and a `friend` graph relation, generates the
SurrealQL DDL, applies it, and then reads and writes a record with full type safety.

## 1. Define the schema

`table(name, shape)` builds a table; each field is an `sz.*` schema, optionally annotated
with SurrealQL metadata via `$`-methods.

```ts twoslash
import { sz, table, relation } from "surreal-zod";
import { surql } from "surrealdb";

export const User = table("user", {
  id: sz.string(),                                  // -> record<user>
  name: sz.string(),
  email: sz.email(),
  status: sz.string().$default("pending"),          // DB-side DEFAULT
  createdAt: sz.datetime().$default(surql`time::now()`).$readonly(),
});

export const Friend = relation("friend", {
  strength: sz.number().$gte(0).$lte(1),            // -> ASSERT $value >= 0 AND $value <= 1
})
  .from(User)
  .to(User);
```

## 2. Generate the DDL

`defineTable` turns a table into a `DEFINE TABLE` + `DEFINE FIELD` script you can send
straight to the database.

```ts twoslash
import { sz, table, defineTable } from "surreal-zod";
import { surql } from "surrealdb";
// ---cut---
const User = table("user", {
  id: sz.string(),
  name: sz.string(),
  email: sz.email(),
  status: sz.string().$default("pending"),
  createdAt: sz.datetime().$default(surql`time::now()`).$readonly(),
});

const ddl = defineTable(User);
```

`ddl` is the SurrealQL string:

```sql
DEFINE TABLE user TYPE NORMAL SCHEMAFULL;
DEFINE FIELD name ON TABLE user TYPE string;
DEFINE FIELD email ON TABLE user TYPE string ASSERT string::is_email($value);
DEFINE FIELD status ON TABLE user TYPE string DEFAULT "pending";
DEFINE FIELD createdAt ON TABLE user TYPE datetime DEFAULT time::now() READONLY;
```

Apply it with the SDK:

```ts twoslash
import { Surreal } from "surrealdb";
declare const db: Surreal;
declare const ddl: string;
// ---cut---
await db.query(ddl);
```

## 3. Write a record

`encode` builds a `CONTENT` payload from your **app** object. DB-filled fields (`id`,
`status`, `createdAt`) are optional — the database fills them — and each value you pass
is validated.

```ts twoslash
import { sz, table } from "surreal-zod";
import { Surreal, surql } from "surrealdb";
declare const db: Surreal;
const User = table("user", {
  id: sz.string(),
  name: sz.string(),
  email: sz.email(),
  status: sz.string().$default("pending"),
  createdAt: sz.datetime().$default(surql`time::now()`).$readonly(),
});
// ---cut---
await db.query(surql`CREATE user CONTENT ${User.encode({
  name: "Ada",
  email: "ada@example.com",
})}`);
```

## 4. Read it back

`decode` turns a raw DB row into your typed app object — `RecordId`, `Date`, and other
codec fields are restored on the way out.

```ts twoslash
import { sz, table } from "surreal-zod";
import { Surreal, surql } from "surrealdb";
declare const db: Surreal;
const User = table("user", {
  id: sz.string(),
  name: sz.string(),
  email: sz.email(),
  status: sz.string().$default("pending"),
  createdAt: sz.datetime().$default(surql`time::now()`).$readonly(),
});
// ---cut---
const [rows] = await db.query<[unknown[]]>(surql`SELECT * FROM user`);
const user = User.decode(rows[0]);
//    ^?

user.createdAt; // a real Date
//   ^?
```

That's the whole loop: **one schema → DDL + validation + a typed read/write mapping.**
